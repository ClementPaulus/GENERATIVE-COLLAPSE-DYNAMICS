These identities do not become "more true" with more tests. A domain with
3 entities and a domain with 406 entities produce the same residuals
because the identities are properties of the kernel function, not
properties of the data.

**Test count measures coverage** — how many entities in a domain's
catalog were analyzed. **Rigor lives in the kernel.** The consciousness
paper (20 systems) has exactly the same structural warranty as the
standard model paper (31 particles) because they both pass the seam
against the same frozen contract with the same frozen parameters.

## Why Every Row Has the Same Residuals

The identity residuals column is not a formality. It is the proof.

Look at the table: **every row is identical in the last column.** The
standard model paper (31 particles, 108 tests) and the consciousness
paper (20 systems, 141 tests) produce the same residuals because they
are feeding different trace vectors through the same kernel function.

The kernel does not know whether its input represents quarks, neurons,
civilizations, or polymers. It computes F, ω, S, C, κ, IC from whatever
trace vector it receives. The three identities hold because they are
**properties of the function**, not properties of the input:

- **F + ω = 1** holds because ω is defined as 1 − F. This is algebraic —
  it cannot fail for any input.
- **IC ≤ F** holds because the geometric mean never exceeds the arithmetic
  mean. This is a theorem about weighted means — it cannot fail for any
  non-negative input.
- **IC = exp(κ)** holds because IC is defined as exp(κ). This is
  definitional — it cannot fail.

A paper with more tests has more **coverage** — it examined more entities
in its domain. A paper with fewer tests examined fewer entities. But the
structural warranty is the same because the warranty comes from the kernel,
not the data.

This is the Cognitive Equalizer principle applied to the corpus itself:
*same kernel + same contract → same warranty, regardless of domain.*
